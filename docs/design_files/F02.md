# F02 — Chess.com URL Support

## Behavior
`POST /api/games` with `{ "url": "<chess.com game URL>" }` fetches the PGN from the chess.com public API, then behaves identically to F01 (validate, parse, persist, return 201). Invalid or non-chess.com URLs return 400.

## Scope
- API gateway only — extends the `POST /api/games` route from F01.
- No new MongoDB fields beyond `source: "chesscom_url"` and `chesscomGameId`.

## URL parsing

Accepted URL patterns:
```
https://www.chess.com/game/live/<gameId>
https://www.chess.com/game/daily/<gameId>
https://chess.com/game/live/<gameId>
https://www.chess.com/analysis/game/live/<gameId>   # analysis view — same gameId
```

Extract the numeric/UUID `gameId` from the path using a regex.

## Chess.com API flow

### Step 1 — fetch game by ID (preferred, direct)
```
GET https://api.chess.com/pub/player/{username}/games/{YYYY}/{MM}
```
That endpoint requires a username — we don't have one from the URL alone.

### Step 2 — use the game endpoint (undocumented but stable for public games)
```
GET https://api.chess.com/pub/game/{gameId}
```
Returns `{ pgn, ... }` directly. Use this first.

If that endpoint returns 404, fall back to searching monthly archives of both players — but that's complex. For v1, if direct fetch fails, return 400 `{ error: "game not found or not public" }`.

### Headers to send
```
User-Agent: personal-chess-analyzer/1.0 (contact: <email>)
```
Chess.com asks bots to identify themselves.

## Input validation
| Condition | Response |
|-----------|----------|
| URL doesn't match chess.com pattern | 400 `{ error: "not a chess.com game URL" }` |
| chess.com API returns non-200 | 400 `{ error: "game not found or not public" }` |
| Fetched PGN fails chess.js validation | 400 `{ error: "invalid PGN from chess.com" }` |
| Network timeout (5s) | 502 `{ error: "chess.com API unavailable" }` |

## MongoDB extras
```js
{
  source: "chesscom_url",
  chesscomGameId: "<extracted id>",   // String; sparse unique index (F08 uses this for dedup)
  ...  // rest same as F01
}
```

## Service layout (`api-gateway/src/services/chesscom.ts`)
```ts
async function fetchPgnFromUrl(url: string): Promise<{ pgn: string; gameId: string }>
```
- Validates URL, extracts gameId
- Calls chess.com API
- Returns raw PGN string + gameId string
