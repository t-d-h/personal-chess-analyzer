# F12: Game Rating & Phase-Specific Accuracy

## Objective
Calculate estimated Game Rating and phase-specific accuracy (Opening, Middlegame, Endgame) for each player during analysis, expose this data through the API, and display it in the frontend's player stats section.

## Backend (C Analyzer)
- **Phase Detection**:
  - *Opening*: Plies 1 to 20, or as long as `is_book` is true.
  - *Endgame*: Parse the `fenAfter` to count the remaining major and minor pieces (Q, R, B, N). If the total count across both colors drops to `≤ 6`, the phase transitions to the Endgame.
  - *Middlegame*: Any move that is not Opening and not Endgame.
- **Phase Accuracy Tracking**: Add variables to `PlayerSummaryStats` to independently track the `accuracy_sum` and `count` for each of the three phases.
- **Estimated Game Rating**: We will add a heuristic function that maps overall accuracy to an estimated Elo rating. For example: `estimatedRating = max(100, round(accuracyPct * 35 - 500))`.
- **JSON Output**: The C worker will append `openingAccuracy`, `midgameAccuracy`, `endgameAccuracy`, and `estimatedRating` to the JSON payload for both White and Black.

## API Gateway
- The API Gateway naturally proxies the JSON payload from MongoDB to the frontend without requiring explicit schema changes.

## Frontend
- **API Interfaces**: Update `frontend/src/services/api.ts` to include the new fields on the `PlayerSummary` interface.
- **PlayerStats.tsx Component**:
  - Add an "Estimated Game Rating" display block next to the overall accuracy percentage.
  - Add a "Phase Review" section rendering a mini report card displaying the accuracy percentage for Opening, Middlegame, and Endgame.
