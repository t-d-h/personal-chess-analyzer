# F15: Scan Chess.com User

## Objective
Support scanning a Chess.com player's games by username from the frontend, listing them in a paginated UI, and allowing the user to select any game to start its analysis.

## Backend (API Gateway)

### 1. Route Definition (`src/api-gateway/src/routes/games.ts`)
Add a new endpoint `GET /api/chesscom/players/:username/games` to fetch a paginated list of completed games for the specified player.
- **Route:** `GET /api/chesscom/players/:username/games`
- **Query Parameters:**
  - `page` (number, default: `1`): The page number.
  - `limit` (number, default: `20`, max: `50`): The page limit.
- **Response Format:**
  ```json
  {
    "games": [
      {
        "url": "https://www.chess.com/game/live/108922485579",
        "uuid": "uuid-string",
        "pgn": "[Event \"Live Chess\"]...",
        "white": { "username": "player1", "rating": 1500, "result": "win" },
        "black": { "username": "player2", "rating": 1480, "result": "resigned" },
        "timeControl": "600",
        "timeClass": "rapid",
        "rules": "chess",
        "playedAt": "2026-06-28T13:00:00.000Z"
      }
    ],
    "hasMore": true
  }
  ```

### 2. Fetching Logic (`src/api-gateway/src/services/chesscom.ts`)
Implement `fetchPlayerGames(username, page, limit)`:
1. Make a request to the Chess.com public API to retrieve all monthly archives:
   `GET https://api.chess.com/pub/player/${username}/games/archives`
2. Reverse the monthly archives array to scan starting from the most recent month.
3. Compute the offset range:
   - `skip = (page - 1) * limit`
   - `targetCount = skip + limit`
4. Iterate through reversed monthly archives:
   - Fetch the archive URL (e.g. `https://api.chess.com/pub/player/${username}/games/${YYYY}/${MM}`).
   - Reverse the games array within the archive to order them by most recent first.
   - Accumulate valid completed chess games (must have PGN).
   - If accumulated games count reaches `targetCount`, stop fetching more archives.
5. Return the sliced games `accumulatedGames.slice(skip, skip + limit)` along with `hasMore`:
   - `hasMore = accumulatedGames.length > targetCount || currentArchiveIndex < archives.length`.
6. Add mock responses when `username` matches:
   - `test-user-mock`: Return 25 mock games (for testing page 1 and page 2).
   - `test-timeout-mock`: Throw a timeout error (`chess.com API unavailable`).
   - `test-notfound-mock`: Throw a player not found error (`player not found`).

## Frontend

### 1. Tab Selector (`frontend/src/pages/Home.tsx`)
Introduce a Tab component to switch between:
1. **Analyze Game** (current PGN/URL inputs form)
2. **Scan Chess.com User** (new UI panel)

### 2. Scan User Panel (`frontend/src/components/PlayerScanner.tsx` or inline in `Home.tsx`)
Create a new panel containing:
- An input for the Chess.com username.
- A "Scan" button.
- A loading skeleton/indicator during fetch.
- An error banner if the request fails.
- A table or card-based list of retrieved games when successful.

Each game row/card should show:
- Date of game (`playedAt`).
- Time Control (e.g. `10 min` or `3+2 Blitz`).
- Opponent and ratings (highlighting White and Black).
- Outcome (e.g. "Won against player2", "Draw", "Lost").
- An **"Analyze"** button.
  - Clicking this triggers `postGame({ url: game.url })` and navigates to `/analysis/:gameId` (or the F14 redirect format if implemented).

### 3. Pagination Controls
Add "Previous" and "Next" buttons below the list:
- "Previous" is disabled if `page === 1`.
- "Next" is disabled if `hasMore === false`.
- Clicking either updates the page state, fetches the new set of games, and scrolls back to the list header.
