# F17 — Dev Docker Compose and Updated Makefile

## Behavior
A separate development-specific Docker Compose file `deploy/dev-docker-compose.yaml` is used to spin up the local development environment containing all five components (`frontend`, `api-gateway`, `analyze-service`, `mongodb`, and `redis`). The root `Makefile` is updated so that `make dev` and `make stop` commands target this new file.

## Scope
- `deploy/dev-docker-compose.yaml` file containing all five services.
- `Makefile` changes updating references to target this compose file.
- `src/api-gateway/Dockerfile` to allow building the api-gateway service.

## deploy/dev-docker-compose.yaml
```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:7
    ports:
      - "27017:27017"
    environment:
      - MONGO_INITDB_DATABASE=chess_analyzer
    volumes:
      - mongodb_data:/data/db
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')"]
      interval: 5s
      timeout: 3s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

  api-gateway:
    build:
      context: ../src/api-gateway
    ports:
      - "18080:18080"
    environment:
      - REDIS_URL=redis://redis:6379
      - MONGO_URL=mongodb://mongodb:27017/chess_analyzer
    depends_on:
      redis:
        condition: service_healthy
      mongodb:
        condition: service_healthy

  analyze-service:
    build:
      context: ../src/analyze-service
    environment:
      - REDIS_URL=redis://redis:6379
      - MONGO_URL=mongodb://mongodb:27017/chess_analyzer
      - STOCKFISH_PATH=stockfish
    depends_on:
      redis:
        condition: service_healthy
      mongodb:
        condition: service_healthy

  frontend:
    build:
      context: ../src/frontend
    ports:
      - "3000:80"
    depends_on:
      - api-gateway

volumes:
  mongodb_data:
  redis_data:
```

## Makefile updates

Update the `dev` target:
```makefile
dev:
	docker compose -f deploy/dev-docker-compose.yaml up -d --build
	@sleep 2
	docker compose -f deploy/dev-docker-compose.yaml exec -T redis redis-cli FLUSHALL
```

Update the `stop` target:
```makefile
stop:
	-kill $$(cat src/api-gateway/.pid) 2>/dev/null || true
	-lsof -t -i :18080 | xargs -r kill -9 2>/dev/null || true
	-rm -f src/api-gateway/.pid
	-kill $$(cat src/frontend/.pid) 2>/dev/null || true
	-lsof -t -i :3000 | xargs -r kill -9 2>/dev/null || true
	-rm -f src/frontend/.pid
	-kill $$(cat src/analyze-service/.pid) 2>/dev/null || true
	-rm -f src/analyze-service/.pid
	-pkill -f analyze-worker 2>/dev/null || true
	-pkill -f stockfish 2>/dev/null || true
	docker compose -f deploy/dev-docker-compose.yaml down
```
