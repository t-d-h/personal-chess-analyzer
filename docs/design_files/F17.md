# F17 — Dev Docker Compose and Updated Makefile

## Behavior
A separate development-specific Docker Compose file `deploy/dev-docker-compose.yaml` is used to spin up the local development database (MongoDB) and cache (Redis). The root `Makefile` is updated so that `make dev` and `make stop` commands target this new file.

## Scope
- `deploy/dev-docker-compose.yaml` file containing the development MongoDB and Redis services.
- `Makefile` changes updating references from the root `docker-compose.yml` to `-f deploy/dev-docker-compose.yaml`.

## deploy/dev-docker-compose.yaml
```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:7
    ports:
      - "27017:27017"
    environment:
      - MONGO_INITDB_DATABASE=chess_analyzer
    volumes:
      - mongodb_data:/data/db
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')"]
      interval: 5s
      timeout: 3s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

volumes:
  mongodb_data:
  redis_data:
```

## Makefile updates

Update the `dev` target:
```makefile
dev:
	docker compose -f deploy/dev-docker-compose.yaml up -d
	@sleep 2
	docker compose -f deploy/dev-docker-compose.yaml exec -T redis redis-cli FLUSHALL
	cd src/analyze-service && make build-worker
	cd src/analyze-service && nohup ./bin/analyze-worker > ../../worker.log 2>&1 & echo $$! > src/analyze-service/.pid
	cd src/api-gateway && nohup npm run dev > dev.log 2>&1 & echo $$! > src/api-gateway/.pid
	cd src/frontend && nohup npm run dev > dev.log 2>&1 & echo $$! > src/frontend/.pid
```

Update the `stop` target:
```makefile
stop:
	-kill $$(cat src/api-gateway/.pid) 2>/dev/null || true
	-lsof -t -i :18080 | xargs -r kill -9 2>/dev/null || true
	-rm -f src/api-gateway/.pid
	-kill $$(cat src/frontend/.pid) 2>/dev/null || true
	-lsof -t -i :3000 | xargs -r kill -9 2>/dev/null || true
	-rm -f src/frontend/.pid
	-kill $$(cat src/analyze-service/.pid) 2>/dev/null || true
	-rm -f src/analyze-service/.pid
	-pkill -f analyze-worker 2>/dev/null || true
	-pkill -f stockfish 2>/dev/null || true
	docker compose -f deploy/dev-docker-compose.yaml down
```
