# F03 — Infra Bootstrap (Docker Compose + Redis + MongoDB)

## Behavior
`docker compose up` brings up Redis and MongoDB. The API gateway connects to both at startup, creates the required indexes on MongoDB, and initializes the Redis consumer group. The service is ready to accept requests that enqueue jobs (XADD) and persist documents.

## Scope
- `docker-compose.yml` at repo root.
- MongoDB index creation on startup.
- Redis consumer group initialization on startup.
- API gateway wired to both — stubbed job enqueue becomes a real `XADD`.
- `make setup`, `make dev` commands in Makefile.

## docker-compose.yml
```yaml
version: "3.9"

services:
  api-gateway:
    build: ./api-gateway
    ports:
      - "8080:8080"
    environment:
      - REDIS_URL=redis://redis:6379
      - MONGO_URL=mongodb://mongodb:27017/chess_analyzer
      - NODE_ENV=development
    depends_on:
      redis:
        condition: service_healthy
      mongodb:
        condition: service_healthy

  analyze-service:
    build: ./analyze-service
    environment:
      - REDIS_URL=redis://redis:6379
      - MONGO_URL=mongodb://mongodb:27017/chess_analyzer
      - STOCKFISH_PATH=/usr/local/bin/stockfish
      - WORKER_CONCURRENCY=2
    depends_on:
      redis:
        condition: service_healthy
      mongodb:
        condition: service_healthy

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

  mongodb:
    image: mongo:7
    ports:
      - "27017:27017"
    volumes:
      - mongo-data:/data/db
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')"]
      interval: 5s
      timeout: 3s
      retries: 5

volumes:
  redis-data:
  mongo-data:
```

## MongoDB indexes (created at API gateway startup)
```js
db.games.createIndex({ pgnHash: 1 }, { unique: true })
db.games.createIndex({ chesscomGameId: 1 }, { unique: true, sparse: true })
db.games.createIndex({ "analysis.status": 1 })
```

## Redis consumer group (created at API gateway startup)
```
XGROUP CREATE chess:analysis-jobs workers $ MKSTREAM
```
(Use `MKSTREAM` so the stream is created even if empty; `$` means workers only see new jobs.)

## Real XADD on POST /api/games
When a new game document is inserted, publish to the stream:
```
XADD chess:analysis-jobs * gameId <objectId> pgn <raw pgn>
```
Store the returned stream entry ID as `jobId` (same as `gameId` for simplicity — just use gameId as jobId since 1:1 mapping).
