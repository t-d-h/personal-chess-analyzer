# F14: Chess.com-style URL for Game Analysis

## Objective
Support Chess.com-style URLs (e.g. `/game/live/170548653608`) on the frontend to display the analysis page, and support looking up games by their Chess.com ID in the backend API gateway.

## Backend (API Gateway)

### 1. Route Lookup (`src/api-gateway/src/routes/games.ts`)
- Modify `GET /api/games/:gameId` and `GET /api/games/:gameId/analysis`.
- Introduce a check for `gameId`. If it matches a standard MongoDB ObjectId (24 hex characters), query the collection by `_id`. Otherwise, query the collection by `chesscomGameId`.
- This ensures that both MongoDB ObjectId hashes and Chess.com IDs can be used interchangeably to fetch game details and analysis results.

### 2. Creation / Ingestion Response (`src/api-gateway/src/routes/games.ts`)
- When a game is successfully ingested (either from a pasted URL or PGN), the `POST /api/games` endpoint currently returns `{ gameId }` (the MongoDB ObjectId).
- We will update the response to also return:
  - `chesscomGameId` (string | null): The Chess.com game ID if it was fetched from a Chess.com URL or if the PGN metadata has a Chess.com link.
  - `gameType` (string | null): The type of Chess.com game, e.g. `"live"` or `"daily"`.

## Frontend

### 1. Route Definitions (`frontend/src/App.tsx`)
- Add a new route mapping for Chess.com game styles:
  ```tsx
  <Route path="/game/:gameType/:gameId" element={<AnalysisPage />} />
  ```

### 2. AnalysisPage Fetching (`frontend/src/pages/AnalysisPage.tsx`)
- The hook or API calls in `AnalysisPage.tsx` should use the `gameId` from the route params.
- Since the API gateway endpoints (`/api/games/:gameId` and `/api/games/:gameId/analysis`) will transparently resolve Chess.com IDs, the page's data-fetching logic will work automatically without modifications.

### 3. Submission Navigation (`frontend/src/pages/Home.tsx` / `frontend/src/components/InputForm.tsx`)
- Upon a successful game submission, the frontend receives the API gateway response containing `gameId`, `chesscomGameId`, and `gameType`.
- If `chesscomGameId` is present, the frontend will navigate to `/game/:gameType/:chesscomGameId` (e.g., `/game/live/170548653608`).
- If `chesscomGameId` is null, it will navigate to the fallback `/analysis/:gameId` using the MongoDB ObjectId.

## Verification
- **Integration Tests**: Add a test in `tests/test_api.py` that ingests a game via Chess.com URL, then fetches the game details and analysis using both the MongoDB ObjectId and the Chess.com game ID to verify dual-lookup.
- **E2E Tests**: Update `tests/test_frontend_e2e.py` to assert that submitting a Chess.com game URL redirects the browser to `http://localhost:3000/game/live/:id` rather than `/analysis/:objectId`.
