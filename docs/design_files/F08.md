# F08 — Caching / Deduplication

## Behavior
`POST /api/games` checks MongoDB for an existing completed analysis document matching the submitted game (by `pgnHash` or `chesscomGameId`) before publishing a new Redis job. A cache hit returns `200` with the existing `gameId` immediately — no new job is created.

## Scope
- API gateway only — modifies the `POST /api/games` handler.
- MongoDB indexes were already created in F03.
- No changes to the C analyze-service.

## Dedup logic (in order)

### For URL submissions (chess.com)
1. Extract `chesscomGameId` from the URL (same as F02).
2. Query: `db.games.findOne({ chesscomGameId })`.
3. If found and `status = "completed"` → return 200 cache hit.
4. If found and `status = "queued" | "running"` → return 200 with existing `jobId` (let client poll).
5. If not found → proceed to insert + enqueue.

### For PGN submissions
1. Compute `pgnHash` (SHA-256 of normalized PGN — strip comments, clock annotations, normalize whitespace).
2. Query: `db.games.findOne({ pgnHash })`.
3. Same 3-branch logic as above.

### Cache hit response
```json
{ "gameId": "<existing id>", "jobId": "<same>", "status": "completed", "cached": true }
```
Use HTTP **200** (not 201) to distinguish from a new submission.

### No hit — insert with duplicate protection
Wrap the MongoDB `insertOne` in a try/catch for `E11000` (duplicate key error):
- If the race condition fires (two concurrent identical submissions), catch the duplicate error and do a `findOne` to return the existing document.
- This makes the endpoint idempotent under concurrency.

## PGN normalization for hashing

```ts
function normalizePgn(pgn: string): string {
  return pgn
    .replace(/\{[^}]*\}/g, '')     // strip { comments }
    .replace(/;[^\n]*/g, '')        // strip ; end-of-line comments
    .replace(/\d+\.\.\./g, '')      // strip black move numbers like "1..."
    .replace(/\[%[^\]]*\]/g, '')    // strip [%clk ...] clock annotations
    .replace(/\s+/g, ' ')           // collapse whitespace
    .trim();
}
```

## Handler pseudocode
```ts
async function handlePostGames(req, reply) {
  const { pgn, url } = req.body;

  // 1. Fetch PGN (F01 / F02)
  let rawPgn: string;
  let chesscomGameId: string | null = null;
  if (url) { ({ pgn: rawPgn, gameId: chesscomGameId } = await fetchPgnFromUrl(url)); }
  else { rawPgn = pgn; }

  // 2. Validate + parse (F01)
  const parsed = parsePgn(rawPgn);   // throws 400 on bad PGN

  // 3. Dedup check
  const pgnHash = sha256(normalizePgn(rawPgn));
  const existing = await findExisting(pgnHash, chesscomGameId);
  if (existing) {
    return reply.code(200).send({ gameId: existing._id, jobId: existing._id, status: existing.analysis.status, cached: true });
  }

  // 4. Insert + enqueue
  const gameId = await insertGame({ rawPgn, pgnHash, chesscomGameId, parsed });
  await enqueueJob(gameId, rawPgn);
  return reply.code(201).send({ gameId, jobId: gameId, status: "queued" });
}
```

## Test plan (`tests/test_cache.py`)

| Test | Expectation |
|------|------------|
| Submit same PGN twice | second call returns 200 with `cached: true`, same gameId |
| Submit same chess.com URL twice | second call returns 200 with `cached: true` |
| Submit same PGN concurrently (10 parallel requests) | all return same gameId, only 1 MongoDB document, no 500 |
| Submit different PGN | different gameId, 201 |
| Submit PGN with comments stripped ≡ previously stored PGN | cache hit |
| Submit chess.com URL, then same game as raw PGN | cache hit on pgnHash |
