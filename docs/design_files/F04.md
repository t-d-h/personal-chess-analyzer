# F04 — Single-Position Stockfish (C analyze-service bootstrap)

## Behavior
A C CLI tool `analyze-single` can spawn a Stockfish subprocess over UCI, send a FEN position, run `go depth 18`, parse the output, and print the centipawn evaluation and best move to stdout. This verifies the UCI pipe integration in isolation before building the full game loop.

## Scope
- C binary only — `analyze-service/` directory.
- No Redis, no MongoDB, no queue.
- Stockfish must be installed on the host (or in Docker via `apt`).
- Verified by `make test-analyze-single`.

## Directory layout
```
analyze-service/
  src/
    uci.c / uci.h      # UCI subprocess management (spawn, write, readline)
    main_single.c      # CLI entry point for F04 test
    analyzer.c / .h    # (skeleton for F05's full game loop)
  Makefile
  Dockerfile
```

## UCI subprocess protocol

### Spawn
```c
// fork + exec stockfish, with two pipes: parent writes to child stdin, reads from child stdout
int sf_spawn(StockfishProc *sf, const char *binary_path);
```

### Initialization sequence
```
→ uci
← uciok
→ isready
← readyok
```

### Analyze one position
```
→ ucinewgame
→ position fen <fen>
→ go depth 18 multipv 2
← info depth ... score cp <N> ... pv <move> ...   (multiple lines)
← bestmove <move> ponder <move>
```

Parse the **last** `info` line before `bestmove` for depth 18 (or whatever the deepest line is).

### Score parsing
- `score cp <N>` → centipawns (already from the side-to-move's perspective)
- `score mate <N>` → cap to ±10 000 cp (positive = mating, negative = mated)
- `multipv 1` line = best move eval; `multipv 2` line = second-best (needed for brilliant detection in F05)

## `uci.h` API
```c
typedef struct {
    int  pid;
    int  stdin_fd;
    int  stdout_fd;
    FILE *out_stream;  // fdopen on stdout_fd for fgets
} StockfishProc;

int  sf_spawn(StockfishProc *sf, const char *path);
void sf_send(StockfishProc *sf, const char *cmd);
// Read lines until a line starting with `prefix` is found; fill buf
int  sf_read_until(StockfishProc *sf, const char *prefix, char *buf, size_t len);
void sf_kill(StockfishProc *sf);

typedef struct {
    int   cp;          // centipawns (capped mate)
    int   is_mate;
    int   mate_in;
    char  best_move[8]; // UCI move string e.g. "e2e4"
    int   depth;
} EvalResult;

int sf_analyze_fen(StockfishProc *sf, const char *fen, int depth, int multipv, EvalResult out[/* multipv */]);
```

## `main_single.c` CLI
```
Usage: analyze-single <fen> [depth]
Output: cp=<N> best=<move>
```
Exit 0 on success, 1 on error.

## Makefile targets (analyze-service/Makefile)
```makefile
CC = gcc
CFLAGS = -Wall -Wextra -O2

build:
	$(CC) $(CFLAGS) src/uci.c src/main_single.c -o bin/analyze-single

test-single:
	./bin/analyze-single "rnbqkbnr/pppppppp/8/8/4P3/8/PPPP1PPP/RNBQKBNR b KQkq e3 0 1" 12
	# Expected: cp value near 0, best move is some reasonable reply
```

## Top-level Makefile target
```makefile
test-analyze-single:
	cd analyze-service && make test-single
```

## Dockerfile (analyze-service)
```dockerfile
FROM ubuntu:24.04
RUN apt-get update && apt-get install -y stockfish gcc make libhiredis-dev libmongoc-dev
WORKDIR /app
COPY . .
RUN make build
CMD ["./bin/analyze-worker"]
```

## Success criteria
Running `make test-analyze-single` prints something like:
```
cp=-15 best=e7e5
```
in under 10 seconds on any modern CPU.

## Dependencies (C)
- POSIX `fork`/`exec`/`pipe` — no external C libs for this feature
- Stockfish binary on `PATH` or `STOCKFISH_PATH` env var
