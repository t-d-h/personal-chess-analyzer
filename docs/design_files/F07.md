# F07 — Remaining API Endpoints

## Behavior
Three additional endpoints on the API gateway:
- `GET /api/jobs/:jobId` — returns live progress from Redis hash.
- `GET /api/games/:gameId` — returns game metadata from MongoDB.
- `GET /api/games/:gameId/analysis` — returns full move list + player summaries once `status=completed`; returns 409 if still running.

## Scope
- API gateway (Node.js/TypeScript/Fastify) only.
- Reads Redis and MongoDB; no writes.

## Endpoint specs

### `GET /api/jobs/:jobId`
Reads `job:<jobId>:progress` hash from Redis.

**Response 200:**
```json
{
  "jobId": "...",
  "status": "queued" | "running" | "completed" | "failed",
  "movesAnalyzed": 14,
  "movesTotal": 38,
  "error": null
}
```

**Response 404:** if the hash key doesn't exist (job unknown or TTL expired).

**Implementation:**
```ts
const data = await redis.hgetall(`job:${jobId}:progress`);
if (!data || Object.keys(data).length === 0) reply.code(404)...
```

---

### `GET /api/games/:gameId`
Reads from MongoDB `games` collection — returns metadata only (no `analysis.moves` array to keep response small).

**Response 200:**
```json
{
  "gameId": "...",
  "source": "pgn_paste" | "chesscom_url",
  "white": { "username": "...", "rating": 1500 },
  "black": { "username": "...", "rating": 1480 },
  "timeControl": "600+0",
  "result": "1-0",
  "ecoCode": "B20",
  "playedAt": "2024-01-15T14:30:00Z",
  "createdAt": "...",
  "status": "completed"
}
```

**Response 404:** if `gameId` is not a valid ObjectId or document not found.

**MongoDB projection:**
```js
{ pgn: 0, "analysis.moves": 0, "analysis.playerSummaries": 0 }
```

---

### `GET /api/games/:gameId/analysis`
Returns full analysis payload.

**Response 200** (only when `analysis.status === "completed"`):
```json
{
  "gameId": "...",
  "moves": [ { "ply": 1, "san": "e4", "classification": "best", ... } ],
  "playerSummaries": [
    { "color": "white", "accuracyPct": 94.2, "acpl": 12, ... },
    { "color": "black", "accuracyPct": 87.1, "acpl": 28, ... }
  ]
}
```

**Response 409** (job in progress):
```json
{ "error": "analysis in progress", "jobId": "...", "status": "running", "movesAnalyzed": 14, "movesTotal": 38 }
```

**Response 404:** game not found.

**Response 500:** status is `"failed"` — include `errorMessage`.

## Route file layout
```
api-gateway/src/routes/
  games.ts       # POST /api/games (F01/F02) + GET /api/games/:id + GET /api/games/:id/analysis
  jobs.ts        # GET /api/jobs/:id
```

## ObjectId validation helper
```ts
function isValidObjectId(id: string): boolean {
  return /^[a-f\d]{24}$/i.test(id);
}
```
Return 404 immediately if invalid — don't let a bad ID reach MongoDB.
