# F01 — PGN Ingestion

## Behavior
`POST /api/games` accepts a JSON body `{ "pgn": "<raw PGN string>" }`, validates it, parses headers, persists a game document to MongoDB with `status: "queued"`, and returns `201 { gameId, jobId, status: "queued" }`. Malformed or missing PGN returns `400`.

## Scope
- API gateway only (Node.js/TypeScript + Fastify).
- No engine, no Redis queue yet (job enqueue is stubbed/logged).
- MongoDB write is real.

## PGN parsing (chess.js)
Use `chess.js` (`Chess.loadPgn()`) for validation — it throws on malformed PGN.

Extract from PGN headers:
| Field | PGN tag | fallback |
|-------|---------|---------|
| white.username | `White` | `"?"` |
| white.rating | `WhiteElo` | `null` |
| black.username | `Black` | `"?"` |
| black.rating | `BlackElo` | `null` |
| timeControl | `TimeControl` | `null` |
| result | `Result` | `"*"` |
| ecoCode | `ECO` | `null` |
| playedAt | `Date` + `StartTime` | `null` |

## pgnHash
SHA-256 of the PGN with comments stripped and whitespace normalized.  
Used later for dedup (F08) — create unique index on `pgnHash` now.

## MongoDB document (initial insert)
```js
{
  source: "pgn_paste",
  chesscomGameId: null,
  pgn: <raw>,
  pgnHash: <sha256>,
  white: { username, rating },
  black: { username, rating },
  timeControl, result, ecoCode, playedAt,
  createdAt: new Date(),
  analysis: {
    status: "queued",
    movesAnalyzed: 0,
    movesTotal: <ply count>,
    errorMessage: null,
    updatedAt: new Date(),
    completedAt: null,
    moves: [],
    playerSummaries: []
  }
}
```

## API response
```json
{ "gameId": "<objectId hex>", "jobId": "<same>", "status": "queued" }
```

## Error responses
| Condition | Status | body |
|-----------|--------|------|
| Missing `pgn` field | 400 | `{ error: "pgn is required" }` |
| Invalid PGN | 400 | `{ error: "invalid PGN: <reason>" }` |
| PGN > 300 plies | 400 | `{ error: "game too long (max 300 plies)" }` |
| Internal error | 500 | `{ error: "internal server error" }` |
