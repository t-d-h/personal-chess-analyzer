# F10 — Hardening

## Behavior
Production-readiness hardening across all layers:
1. **API gateway** — IP-based rate limiting on `POST /api/games`; reject PGNs exceeding 300 plies or 512KB.
2. **C analyze-service** — per-move Stockfish timeout (5s) with automatic process restart; per-job overall timeout (5 min) that marks the job `failed` and calls `XACK`.

## Scope
- API gateway: rate limiting + input size guards (minor additions to F01 handler).
- C analyze-service: timeout logic in `uci.c` and `worker.c`.
- No new endpoints; no frontend changes.

## 1. API gateway — rate limiting

Use `@fastify/rate-limit`:
```ts
await app.register(import('@fastify/rate-limit'), {
  max: 10,                  // 10 requests
  timeWindow: '1 minute',   // per IP per minute
  keyGenerator: (req) => req.ip,
  errorResponseBuilder: () => ({
    error: 'Too many requests',
    retryAfter: 60
  })
});
// Applied globally; Fastify only applies to registered routes.
// Attach specifically to POST /api/games for stricter control.
```

Rate limit applies **only** to `POST /api/games` (the expensive endpoint).

## 2. API gateway — input size guards

### PGN size limit
```ts
if (Buffer.byteLength(pgn, 'utf8') > 512 * 1024) {
  return reply.code(400).send({ error: 'PGN too large (max 512KB)' });
}
```
Add `bodyLimit` to Fastify options: `bodyLimit: 600 * 1024` (600KB raw body limit so the framework itself doesn't reject first).

### Ply count limit (already in F01, enforce here explicitly)
```ts
const chess = new Chess();
chess.loadPgn(pgn);
if (chess.history().length > 300) {
  return reply.code(400).send({ error: 'game too long (max 300 plies)' });
}
```

## 3. C analyze-service — per-move Stockfish timeout (5s)

Replace blocking `fgets` in `sf_read_until` with `poll()`-based read:

```c
int sf_read_line_timeout(StockfishProc *sf, char *buf, size_t len, int timeout_ms) {
    struct pollfd pfd = { .fd = sf->stdout_fd, .events = POLLIN };
    int ready = poll(&pfd, 1, timeout_ms);
    if (ready <= 0) return -1;  // timeout or error
    return (fgets(buf, len, sf->out_stream) != NULL) ? 0 : -1;
}
```

In `sf_analyze_fen`: if `sf_read_line_timeout` returns -1 (timeout waiting for `bestmove`):
1. `sf_kill(sf)` — SIGKILL the subprocess.
2. `sf_spawn(sf, path)` — restart fresh Stockfish.
3. Re-send `uci` + `isready` handshake.
4. Record the move's eval as `NULL` / `evalCpPlayed = INT_MIN` (sentinel).
5. Continue to the next move — don't fail the whole job.

## 4. C analyze-service — per-job overall timeout (5 min)

Record `job_start = time(NULL)` before the move loop. After each move:
```c
if (time(NULL) - job_start > JOB_TIMEOUT_SECS) {
    // mark remaining moves as NULL
    // write partial result to MongoDB with status = "failed"
    // set errorMessage = "job timed out after 5 minutes"
    // HSET job:<gameId>:progress status failed
    // XACK the stream entry
    goto job_done;
}
```
`JOB_TIMEOUT_SECS` = 300, configurable via `JOB_TIMEOUT` env var.

## 5. Env vars added

| Variable | Default | Description |
|----------|---------|-------------|
| `JOB_TIMEOUT` | `300` | Per-job hard deadline (seconds) |
| `MOVE_TIMEOUT_MS` | `5000` | Per-move Stockfish response deadline (ms) |
| `RATE_LIMIT_MAX` | `10` | Max POST /api/games per window |
| `RATE_LIMIT_WINDOW` | `60000` | Rate limit window (ms) |

## Test plan (`tests/test_hardening.py`)

### Rate limiting
| Test | Expectation |
|------|------------|
| Send 11 POST /api/games in 1 minute from same IP | 11th returns 429 |
| Send 10 from IP-A and 10 from IP-B | all 20 succeed (limit is per-IP) |
| Wait 1 minute, try again | succeeds |

### Input size
| Test | Expectation |
|------|------------|
| Submit PGN with 301 plies | 400 |
| Submit 513KB body | 400 |
| Submit valid 300-ply PGN | 201 |

### Timeouts (integration — mocked Stockfish)
For timeout tests, replace Stockfish with a mock binary that:
- Hangs on a specific move (sleep forever) → expect move recorded as NULL, job continues.
- Hangs on all moves → expect job fails with `"job timed out"` within 5.5 min.

| Test | Expectation |
|------|------------|
| Mock Stockfish hangs on move 5 of 20 | job completes, move 5 has null eval, moves 6-20 have evals |
| Mock Stockfish hangs on all moves | job fails with errorMessage="job timed out after 5 minutes" |
| Redis progress after timeout | status=failed, XPENDING=0 |
